# Plan: Close issue #14 — write path, visibility, multilingual router, /memory-augment
Status: completed
Started: 2026-05-27

## Context

Issue #14 documents 6 gaps between ADR-0007/ADR-0008's promise and the
live state of the system. PR #13 (ADR-0008) refined the per-node
schema and consolidation prompt but did not address the **write path**
(no propagation from `/plan`/`/adr`/`/done` into node bodies), nor
**visibility** into pipeline state. The router is English-only, the
SessionStart hook lists superseded ADRs without status, and
`/memory-augment` is referenced but does not exist.

External evidence cited in issue #14:
- 13/13 nodes had empty body sections (pre-ADR-0008 migration).
- `.claude/memory/.last-consolidated` absent.
- Router pattern-matches English regexes only; Hebrew prompts fall
  through to silent suppression.
- SessionStart lists ADR-0006 (superseded) with no marker.

## Goal

After this plan:
(a) A plan closed via `/done` propagates its invariants/decisions
    into matching nodes (those whose `code:` paths overlap files
    touched by the plan).
(b) The user sees pipeline state at a glance — SessionStart line plus
    a dedicated `doctor.sh` script.
(c) The router presents a menu for non-English prompts long enough to
    be actionable.
(d) `/memory-augment` exists as a first-class command.
(e) SessionStart skips/marks superseded ADRs.

## Options considered

- **A — six separate PRs.** Rejected: most gaps share files
  (`consolidate.sh`, `/done`, `session-start.sh`); split creates
  needless merge conflicts.
- **B — one PR covering all six gaps (chosen).** Schema work is in
  ADR-0008; this plan implements remaining write/visibility pieces.
  Commits are incremental so each stands alone for review.
- **C — write a new ADR first.** Rejected: no new architectural
  decision. ADR-0008's "node populated by consolidation + cold-start
  scan" is simply extended with one more source (plan/ADR text at
  `/done` time).

## Decision

Option B. Low-risk gaps first, propagation last (the largest).

## Steps

1. **Gap #4 — SessionStart ADR status.** In
   `templates/hooks/session-start.sh` (+ dogfood): when listing
   recent ADRs, awk-extract `^Status:` from each file; skip
   `superseded`/`deprecated` and suffix `(proposed)` on non-accepted.

2. **Gap #5 — router multilingual fallback.** In
   `templates/hooks/prompt-submit.sh` (+ dogfood): if no regex
   matched and `${#prompt}` >= 40 and the prompt is not obviously
   pasted code (no triple-backtick fence), set `intent="ambiguous"`
   and emit a generic menu via the existing context-injection path.

3. **Gap #2 — `/remember --section [--node <path>] <text>`.** In
   `templates/commands/remember.md` (+ dogfood): (a) parse the two
   flags; (b) update the 4-section list to ADR-0008's six sections;
   (c) if `--section` given, skip auto-classify; (d) if `--node`
   given, skip leaf-pick.

4. **Gap #3 — `aims doctor` + SessionStart pipeline line.** New
   `templates/memory/doctor.sh` (+ dogfood): prints node count, dirty
   count, `.last-consolidated` mtime ("never" if absent), API key
   presence (yes/no — never the value), lint summary, count of nodes
   >4 KB. Update `session-start.sh` to emit a one-liner derived from
   the same data. Update `done.md` to call `doctor.sh` in the final
   report.

5. **Gap #1 — `/done` propagation bridge.** Extend
   `templates/memory/consolidate.sh` (+ dogfood) to read
   `AIMS_EXTRA_CONTEXT` env var and fold it into the prompt under
   "ADDITIONAL CONTEXT (plan/ADR text driving this consolidation)".
   In `templates/commands/done.md` (+ dogfood) add a step before the
   throttled consolidation pass: `git diff --name-only
   <plan-start>..HEAD`, then for each node whose `code:` overlaps,
   collate plan body + ADR bodies into `AIMS_EXTRA_CONTEXT` and call
   `consolidate.sh` directly on those nodes.

6. **Gap #6 — `/memory-augment`.** New
   `templates/commands/memory-augment.md` (+ dogfood). Two modes:
   `add tag <name>` (scan a source subdir, propose nodes under the
   new tag) and `add node <path>` (scaffold one node for a specified
   code path). Uses existing `new-node.sh` + `lint.sh`. Update
   `templates/commands/memory-init.md` to point users at this
   command instead of saying "a future /memory-augment will…".
   Update `commands/init-workflow.md` (+ dogfood) to copy the new
   command file into TARGET.

7. **Sync + tests + commit.** Per-step `bash -n` on all touched
   scripts; per-step dogfood mirroring (`templates/...` →
   `.claude/...`). One commit per logical step (six commits) so
   review can move incrementally. Push to
   `claude/stoic-goodall-ScbPt`.

8. **Close #14.** PR body lists each gap → commit mapping; PR
   description includes `Fixes #14`.

## Verification

- `bash -n templates/hooks/*.sh templates/memory/*.sh .claude/hooks/*.sh .claude/memory/*.sh` — passes.
- `bash .claude/memory/lint.sh` — clean.
- **Gap 4:** running `bash .claude/hooks/session-start.sh` does not
  list ADR-0006, or lists it with `(superseded)` suffix.
- **Gap 5:** invoking the router with a 50+ char Hebrew prompt sets
  `intent="ambiguous"` (verified by trace flag) and emits the menu.
- **Gap 2:** `/remember --section "Known issues" --node
  docs/memory/installer/init-workflow.md "open: test"` adds exactly
  one bullet under `## Known issues`; reverted after.
- **Gap 3:** `bash .claude/memory/doctor.sh` prints all six fields;
  SessionStart includes the one-liner.
- **Gap 1:** with `ANTHROPIC_API_KEY` set, `/done` on the current
  plan populates at least one node body section that was empty
  before. Without the key, `/done` warns explicitly that propagation
  was skipped.
- **Gap 6:** `/memory-augment add node test/scratch` scaffolds a
  valid six-section node; lint stays clean; cleanup before commit.

## Risks / unknowns

- Gap 1 needs `ANTHROPIC_API_KEY` to actually write content; without
  it, `consolidate.sh` exits 0 silently. Mitigation: doctor.sh
  surfaces this; `/done` warns explicitly.
- Gap 5 false positives on long code paste-ins. Mitigation: skip if
  the prompt contains a triple-backtick fence or exceeds ~2 KB.
- PR size: if review feedback flags it as too large, split Gap 1 and
  Gap 6 into a follow-up PR.
- Gap 1 propagation may add weak content if plan and node only
  loosely overlap. Mitigation: prompt instructs "do not invent; if
  the connection is weak, add nothing".

## ADRs to record after implementation

- None. ADR-0008's "consolidation + cold-start scan populate node
  sections" is extended with one more source (plan/ADR text at
  `/done`); the architectural decision is unchanged.
- If implementation reveals a new design decision (e.g. a policy on
  *when* to skip propagation), write an ADR then.

## Outcome

All six gaps from issue #14 implemented in one PR, no new ADRs needed.

- Gap 4 (SessionStart ADR status): session-start.sh now reads each
  ADR's `Status:` and skips superseded/deprecated, suffixes others
  with their status when not accepted.
- Gap 5 (router multilingual fallback): prompt-submit.sh classifies
  unmatched prompts of 40–2048 chars (no fenced code) as
  `intent=ambiguous` and emits a generic workflow menu.
- Gap 2 (`/remember --section --node`): remember.md argument
  grammar parses both flags; section list updated to the six
  ADR-0008 sections.
- Gap 3 (doctor + SessionStart line): new `templates/memory/doctor.sh`
  (multi-line + `--brief`); SessionStart appends the brief line;
  `/done` calls the full report.
- Gap 1 (`/done` propagation bridge): `consolidate.sh` reads
  `AIMS_EXTRA_CONTEXT`; `/done` collates plan body + new ADRs and
  exports it for the forced consolidation pass.
- Gap 6 (`/memory-augment`): new command with two modes — `add tag`
  and `add node`; `/memory-init` updated to point at it;
  `init-workflow.md` (both root + dogfood) copies it on install.
